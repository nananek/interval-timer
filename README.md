# インターバルタイマー

短答式の練習用インターバルタイマー。本番のように「総残り時間」を見ながらペース配分する練習ができます。各問は「解く時間」で強制的に区切られ、続けて「解説を読む時間」を計測し、指定した問題数ぶん繰り返します。GitHub Pages にそのまま置ける PWA（インストール可能・オフライン対応）です。

## 特徴

- **総残り時間で本番想定**：画面の主役は「総残り時間」（解く時間 × 問題数。例：2 分 × 10 問 = 20:00）。**解く間だけ**減り、各問は解く時間が経過すると強制的に解説へ切り替わります。解説中は総残り時間が淡色で凍結し、リングの下に解説用のカウントダウンが出ます。本番で残り時間を見ながら時間管理する感覚を練習できます。2 分経過前に「次へ」で切り上げた余り時間は繰り越さず、各問つねに解く時間ぶんを消費した扱いになります。
- **解く → 解説のインターバル**：1 問あたり「解く」と「解説」の 2 フェーズ。フェーズごとに色（解く=オレンジ / 解説=緑）と背景が変わります。
- **問題数を指定**：設定した問題数を終えると自動で完了します。
- **進捗が一目でわかる**：リング内の「問題 N / 全数」表示、右上のカウンタ、問題数ぶんのドット列（完了=淡色 / 現在=点灯）。
- **時間・問題数を自由に変更**：設定シートから解く時間・解説時間・問題数を変更でき、内容は端末に保存されます。±ボタンの微調整に加え、各入力をタップすると **テンキー** で直接入力できます（時刻は分秒で右詰め入力）。
- **切替の通知**：フェーズ切替時にビープ音（解く=下降音 / 解説=上昇音）・バイブ・OS 通知。それぞれ個別にオン/オフ可能（通知は許可が要るためオプトイン）。
- **PWA**：ホーム画面に追加でき、オフラインでも動作します。アイコン・絵文字（Twemoji）はすべて埋め込み済みで外部リクエストはありません。
- **アプリ更新の自動検知**：新しいバージョンを配布すると、起動・復帰時にこれを検知して画面上部に「更新」バーを表示します。タップすると最新版に切り替わります（稼働中のタイマーを勝手に中断しません）。

## 操作

| 操作 | 内容 |
|------|------|
| 開始 / 一時停止 / 再開 | 中央のボタン（PC ではスペースキー） |
| 次のフェーズへスキップ | ⏭ ボタン（PC では → キー） |
| 設定 | ⚙ ボタン（時間・問題数・通知） |

## 使い方（ローカル）

PWA の Service Worker は `file://` では動かないため、簡易サーバー経由で開きます。

```sh
# どれか一つ
python3 -m http.server 8000
# または
npx serve .
```

ブラウザで `http://localhost:8000` を開きます。

## GitHub Pages への公開

1. このリポジトリを GitHub に push する。
2. **Settings → Pages → Source** を「Deploy from a branch」にし、ブランチを `main` / `(root)` に設定。
3. 数十秒後、`https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます。
4. スマホのブラウザで開き、メニューまたは設定シート内の「ホーム画面に追加」からインストール。

パスはすべて相対参照（`./`）なので、サブディレクトリ配置（リポジトリ名が URL に入る形）でも動作します。

## バックグラウンド動作・通知・暗転抑制について

タイマーは経過時間（絶対時刻）ベースで動くため、タブが裏に回ったり画面がロックされて
`setInterval` が間引かれても、復帰した瞬間に実時間ぶん正しく補正され、跨いだフェーズは
自動で追いつきます。稼働中は Screen Wake Lock API（対応ブラウザ）で画面の暗転を抑制し、
一時停止・完了で解除します。通知は「音（WebAudio）＋バイブ（`navigator.vibrate`）＋
OS 通知（Notifications API、設定でオプトイン）」の多層構成です。ただしブラウザの制約上、
**バックグラウンドで指定時刻に確実に鳴らす予約通知や、サーバーを使う Push 通知は
（静的ホスティングの GitHub Pages では）対応しません**。OS 通知はタブが生きていてフェーズ
切替を検知できたタイミングで発火するもので、定刻保証はありません。

## 補足

- iOS / Safari はバイブ API（`navigator.vibrate`）と Screen Wake Lock の一部に非対応／制限があり、iPhone では音のみ・暗転抑制なしに縮退することがあります。Android Chrome では音＋バイブ＋暗転抑制が動作します。各機能は機能検出でガードし、非対応環境では静かに無効化されます。
- 内容を更新したら、`sw.js` の `CACHE` のバージョン文字列（例：`tanto-iv-v5`）を上げてデプロイします。HTML は network-first で配信し、Service Worker は新 SW を即時有効化せず待機させるため、起動・復帰時の更新チェックで検知され、画面上部の「更新」バー → タップで安全に切り替わります（稼働中のタイマーは中断しません）。すべてのタブ／PWA を閉じて開き直した場合は自動的に新版が有効化されます。
